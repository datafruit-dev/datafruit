## Authentication

All endpoints require:

- `api_key` ‚Äì your API access key
- `project_id` ‚Äì your UUID identifying the active project

If either is invalid or missing, you'll receive a `401 Unauthorized` error.

## üì¶ Create and Deploy App

Deploy a GitHub project as a Kubernetes app.

### Endpoint

```http
POST /create_and_deploy_app_api
```

### Description

Initiates a background deployment using the specified GitHub repository and command.

### Request Body

```json
{
  "api_key": "string",
  "project_id": "string (UUID)",
  "name": "string",
  "github_token": "string",
  "repo_url": "string",
  "repo_branch": "string",
  "start_command": "string"
}
```

### Response

```json
{
  "message": "Deployment initiated"
}
```

### Errors

- `400 Bad Request` ‚Äì Invalid GitHub URL/token or project inactive
- `500 Internal Server Error` ‚Äì Deployment error

## üóëÔ∏è Delete App

Delete a deployed application.

### Endpoint

```http
POST /delete_app_api
```

### Description

Initiates a background task to delete the specified application from the cluster.

### Request Body

```json
{
  "api_key": "string",
  "project_id": "string (UUID)",
  "name": "string"
}
```

### Response

```json
{
  "message": "Deletion initiated"
}
```

### Notes

- This operation is asynchronous.
- The endpoint **does not** wait for confirmation of deletion.
- Check the status afterward using `/get_app_status_api`.

### Errors

- `401 Unauthorized` ‚Äì Invalid API key
- `500 Internal Server Error` ‚Äì Deletion error

## üìä Get App Status

Check the deployment status of an app.

### Endpoint

```http
POST /get_app_status_api
```

### Description

Returns the current deployment status and service URL (if active).

### Request Body

```json
{
  "api_key": "string",
  "project_id": "string (UUID)",
  "name": "string"
}
```

### Response

```json
[
  "active" | "creating" | "failed" | "not_created",
  "string | null" // service_url
]
```

### Errors

- `401 Unauthorized` ‚Äì Invalid API key
